## Subcommands

### `lexicon gate init`

Initialize the default gates configuration.

```bash
lexicon gate init
```

Creates `specs/gates.toml` with four default gates:

| Gate       | Command                        | Category | Timeout | Skippable |
|------------|--------------------------------|----------|---------|-----------|
| fmt        | `cargo fmt -- --check`         | Required | 60s     | No        |
| clippy     | `cargo clippy -- -D warnings`  | Required | 120s    | No        |
| unit-tests | `cargo test`                   | Required | 300s    | No        |
| doc-tests  | `cargo test --doc`             | Scored   | 120s    | Yes       |

On success:

```
Gates initialized
```

## Customizing Gates

After initialization, edit `specs/gates.toml` to add or modify gates. Each gate entry requires:

```toml
[[gates]]
id = "my-gate"
label = "My Custom Gate"
command = "my-command --flag"
category = "scored"       # required | scored | advisory
timeout_secs = 120
allow_skip = true
```

### Adding Gates

Append new `[[gates]]` entries for project-specific checks:

```toml
[[gates]]
id = "deny-unsafe"
label = "No Unsafe Code"
command = "cargo deny check unsafe"
category = "required"
timeout_secs = 60
allow_skip = false
```

### Removing Gates

Remove the `[[gates]]` entry. Be aware that removing a required gate or changing its category is considered a weakening and will be flagged if `gate_weakening_requires_approval` is enabled in the manifest policy.

See [Gates Concepts](/concepts/gates/) for details on categories, skip policy, and weakening detection.
