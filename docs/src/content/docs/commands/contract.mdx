## Subcommands

### `lexicon contract new`

Create a new contract interactively.

```bash
lexicon contract new
```

Launches a conversational workflow that guides you through defining:

- Contract ID (slug format, e.g., `key-value-store`)
- Title and scope description
- Invariants with severity levels
- Required semantics with test tags
- Forbidden semantics with test tags
- Edge cases with expected behavior
- Usage examples

The contract is saved to `specs/contracts/<id>.toml` with `draft` status and `experimental` stability.

On success:

```
New Contract
...
Contract 'key-value-store' created
```

If cancelled:

```
Contract creation cancelled
```

### `lexicon contract list`

List all contracts in the repository.

```bash
lexicon contract list
```

Scans `specs/contracts/` for TOML files and lists their IDs.

```
Contracts
  key-value-store
  parser
  auth-handler
```

If no contracts exist:

```
No contracts found. Run `lexicon contract new` to create one.
```

### `lexicon contract lint`

Lint contracts for potential issues.

```bash
lexicon contract lint
```

Checks contracts for structural problems such as missing fields, duplicate IDs, or orphaned references.

:::note
Contract linting is not yet fully implemented in the current version.
:::

### `lexicon contract generate`

AI-generate a contract from a natural language description.

```bash
lexicon contract generate "<intent>"
```

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon contract generate "async key-value store with TTL support"
```

Generates a complete TOML contract with invariants, required and forbidden semantics, edge cases, and examples. The generated contract is shown as a patch preview -- you review and accept or reject before it is written to `specs/contracts/`.

```
Patch Preview
  File: specs/contracts/async-key-value-store-with-ttl-support.toml
────────────────────────────────────────────
id = "async-key-value-store-with-ttl-support"
title = "Async Key-Value Store with TTL Support"
...
────────────────────────────────────────────
? What would you like to do?
> Accept
  Reject
```

See [`lexicon generate`](/commands/generate) for more on AI artifact generation.

### `lexicon contract infer`

Infer a contract from the public API surface.

```bash
lexicon contract infer
lexicon contract infer --path crates/my-crate/src
```

Scans the public API (traits, methods, structs, error types, documentation) and has AI propose a draft contract based on observed behavior. Useful for bootstrapping contracts on existing codebases.

Requires authentication (`lexicon auth login`).

| Flag | Description |
|------|-------------|
| `--path <dir>` | Source directory to scan (defaults to `src/`) |

**Example:**

```bash
lexicon contract infer
```

```
Infer Contract from API
  Scanning public API surface...
────────────────────────────────────────────

Patch Preview
  File: specs/contracts/inferred-contract.toml
────────────────────────────────────────────
id = "inferred-contract"
title = "Inferred Contract"
...
────────────────────────────────────────────
? What would you like to do?
> Accept
  Edit
  Reject
```

The inferred contract is a starting point -- use **Edit** to refine the AI's suggestions before accepting.
