## Subcommands

### `lexicon conformance add`

Add a conformance suite to a contract.

```bash
lexicon conformance add <contract_id>
```

Generates a test harness in `tests/conformance/` that verifies an implementation satisfies the contract's invariants and required semantics.

### `lexicon conformance sync`

Re-sync conformance suites when contracts change.

```bash
lexicon conformance sync
```

### `lexicon conformance generate`

AI-generate conformance tests from a natural language description.

```bash
lexicon conformance generate "<intent>"
```

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon conformance generate "cache trait with async get/set/delete"
```

Generates a Rust conformance test file with:

- Trait-based test harness
- Tests for all invariants and required semantics
- Edge case tests
- Test tags as comments (`// lexicon::tag("...")`)

The generated file is shown as a patch preview. You review and accept or reject before it is written to `tests/conformance/`.

### `lexicon conformance from-contract`

Generate conformance tests from an existing contract's clauses.

```bash
lexicon conformance from-contract <contract_id>
```

Unlike `generate` which takes free-text intent, `from-contract` reads a parsed contract TOML and uses its structured data (invariants, required semantics, forbidden semantics, edge cases) to produce targeted tests.

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon conformance from-contract key-value-store
```

### `lexicon conformance property`

Generate property-based tests from a contract's invariants using the `proptest` crate.

```bash
lexicon conformance property <contract_id>
```

Reads a contract's invariants and generates `proptest!` macro-based tests that verify properties hold for arbitrary inputs. Best used when your contract has invariants like "insert then get returns same value" that should hold for all inputs.

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon conformance property key-value-store
```

Generated tests use `prop_assert!` and `prop_assert_eq!` assertions and are written to `tests/property/`.

### `lexicon conformance fuzz`

Generate a fuzz test harness from a contract.

```bash
lexicon conformance fuzz <contract_id>
```

Produces a `libfuzzer-sys` fuzz target that exercises contract invariants with arbitrary input. The generated harness uses `#![no_main]` and `fuzz_target!` and is written to `fuzz/fuzz_targets/`.

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon conformance fuzz key-value-store
```

### `lexicon conformance edge-cases`

Generate targeted edge case tests from a contract.

```bash
lexicon conformance edge-cases <contract_id>
```

Extracts the `edge_cases` array from a contract and generates one test per edge case, focusing on boundary conditions, error paths, and unusual inputs. Generated tests are written to `tests/edge_cases/`.

Requires authentication (`lexicon auth login`).

**Example:**

```bash
lexicon conformance edge-cases key-value-store
```

```

```
