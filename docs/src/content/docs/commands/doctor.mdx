## Usage

```bash
lexicon doctor
```

## Description

Runs a series of health checks on the lexicon-managed repository and reports the status of each component.

## Health Checks

The doctor command checks the following:

### 1. Manifest

Verifies that `.lexicon/manifest.toml` exists. If missing, the remaining checks are skipped and the command advises running `lexicon init`.

### 2. Contracts

Counts the number of contracts in `specs/contracts/`. Reports the count.

### 3. Scoring Model

Checks whether `specs/scoring/model.toml` exists. If missing, advises running `lexicon score init`.

### 4. Gates

Checks whether `specs/gates.toml` exists. If missing, advises running `lexicon gate init`.

### 5. CLAUDE.md

Checks whether `CLAUDE.md` exists in the repository root. If missing, advises running `lexicon sync claude`.

## Example Output

A healthy repository:

```
Doctor — checking repo health
 ✓ Manifest found
 ℹ 3 contract(s)
 ✓ Scoring model configured
 ✓ Gates configured
 ✓ CLAUDE.md present
```

A freshly initialized repository with no configuration:

```
Doctor — checking repo health
 ✓ Manifest found
 ℹ 0 contract(s)
 ⚠ No scoring model — run `lexicon score init`
 ⚠ No gates — run `lexicon gate init`
 ⚠ No CLAUDE.md — run `lexicon sync claude`
```

An uninitialized repository:

```
Doctor — checking repo health
 ✗ No manifest — run `lexicon init` first
```
