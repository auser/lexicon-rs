The `workspace` command group manages architecture governance for Cargo workspaces.

## `lexicon workspace init`

Initialize workspace governance. Requires `lexicon init` to have been run first.

```bash
lexicon workspace init
```

This command:
1. Detects Cargo workspace members from your `Cargo.toml`
2. Auto-assigns crate roles based on naming conventions (e.g., `-cli` → Application, `-spec` → Foundation)
3. Creates `.lexicon/workspace.toml` with crate roles and shared contract references
4. Creates `.lexicon/architecture/rules.toml` with default dependency rules

### Default Dependency Rules

The default rules enforce a layered architecture:

| From Role    | Allowed Dependencies               | Forbidden Dependencies        |
|-------------|-------------------------------------|-------------------------------|
| Foundation  | Utility                             | Application, Adapter          |
| Application | Foundation, Interface, Adapter, Utility | Test                      |
| Interface   | Foundation, Utility                 | Application, Adapter, Test    |
| Adapter     | Foundation, Interface, Utility      | Application, Test             |

### Crate Role Inference

Roles are inferred from crate names:

| Name Contains | Assigned Role |
|--------------|---------------|
| `cli`, `bin`, `app` | Application |
| `spec`, `types`, `model` | Foundation |
| `api`, `trait`, `interface` | Interface |
| `adapter`, `driver`, `plugin` | Adapter |
| `test`, `fixture`, `mock` | Test |
| `util`, `common`, `helper`, `fs` | Utility |
| (other) | Foundation |

Edit `.lexicon/workspace.toml` to adjust roles after initialization.

## `lexicon workspace verify`

Verify workspace architecture against defined rules.

```bash
lexicon workspace verify
```

Checks:
- All workspace members have assigned roles
- Architecture rules file exists
- Workspace manifest is valid

## `lexicon workspace doctor`

Check workspace health and report issues.

```bash
lexicon workspace doctor
```

Reports on:
- Missing workspace manifest
- Missing architecture rules
- Missing architecture directory
- Workspace members without assigned roles
