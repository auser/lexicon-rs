## Subcommands

### `lexicon score init`

Initialize the default scoring model.

```bash
lexicon score init
```

Creates `specs/scoring/model.toml` with the default six-dimension scoring model:

| Dimension              | Weight | Category | Source     |
|------------------------|--------|----------|------------|
| Correctness            | 30     | Required | Gate       |
| Conformance Coverage   | 25     | Scored   | Test Suite |
| Behavior Pass Rate     | 15     | Scored   | Test Suite |
| Lint Quality           | 10     | Scored   | Gate       |
| Documentation Complete | 10     | Advisory | Manual     |
| Panic Safety           | 10     | Scored   | Gate       |

Default thresholds: pass at 80%, warn at 60%.

On success:

```
Scoring model initialized
```

### `lexicon score explain`

Display the current scoring model with explanations.

```bash
lexicon score explain
```

Reads `specs/scoring/model.toml` and prints a human-readable explanation of each dimension, its weight, category, source, and how thresholds work.

## Customizing the Model

After initialization, edit `specs/scoring/model.toml` directly to:

- Adjust weights (they do not need to sum to 100, but it is conventional)
- Change categories (required/scored/advisory)
- Add project-specific dimensions
- Tune pass/warn thresholds

See [Scoring Concepts](/concepts/scoring/) for details on how scoring works.
